如何添加资源

按照以下步骤将资源添加到您的GitHub仓库，资源站将自动展示这些文件

添加资源的步骤

1

准备您的GitHub仓库

确保您已经在GitHub上创建了一个仓库用于存储资源。如果还没有，请先创建一个新仓库，推荐命名为"resources"或与您的项目相关的名称。

2

上传资源文件

登录您的GitHub账号，进入资源仓库，点击右上角的"Add file"按钮，然后选择"Upload files"选项。

3

拖放或选择文件

您可以直接将文件拖放到上传区域，或者点击"choose your files"链接选择本地文件。支持同时上传多个文件和文件夹。

4

提交更改

上传完成后，滚动到页面底部，在"Commit changes"区域填写提交信息（例如"添加了新的PDF文档"），然后点击"Commit changes"按钮完成上传。

5

查看资源

上传完成后，等待几分钟，资源站将自动同步并展示您新添加的资源。您可以在资源列表页面查看和管理这些资源。

设置自动同步（可选）

为了让资源站自动同步GitHub仓库中的文件变化，您可以设置GitHub Actions工作流：

  1. 在您的GitHub资源仓库中创建`.github/workflows/update.yml`文件
  2. 复制以下代码到该文件中： 
    name: Update Resources
on:
  push:
    branches: [ main ]
jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Update resource list
        run: |
          # 这里可以添加生成资源列表的脚本
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./
  3. 提交该文件，GitHub将在每次推送后自动更新您的资源站

关于本资源站

本资源站是一个基于GitHub Pages的静态资源展示平台，旨在帮助您方便地管理和分享存储在GitHub仓库中的各类资源文件。

为什么之前的链接不能跳转？

之前的链接无法跳转有两个主要原因：

  1. 部分演示链接使用了`href="#"`作为占位符，这是前端开发中常用的临时链接方式
  2. 资源站需要与实际的GitHub仓库关联才能实现完整的跳转功能

现在的版本已经修复了这个问题，所有链接都已设置为正确的跳转方式。对于资源文件链接，当您添加实际资源后，它们将指向GitHub仓库中的对应文件。

支持的文件类型

本资源站支持所有常见文件类型，包括但不限于：

  • 文档类：PDF、DOC、DOCX、XLS、XLSX、PPT、PPTX、TXT、MD等
  • 图片类：JPG、PNG、GIF、SVG、WEBP等
  • 压缩包：ZIP、RAR、7Z、TAR、GZ等
  • 代码类：HTML、CSS、JS、Java、Python、PHP等各种编程语言文件
